# DroidLink: Electron → Tauri 迁移计划

## 目标

将 Electron（~150MB）迁移为 Tauri v2（~10MB），渐进式迁移，先做核心功能。

---

## Phase 1: 项目初始化

在现有项目根目录下创建 Tauri 子目录，保留 Electron 代码不动。

**创建文件：**

- `src-tauri/Cargo.toml` — Rust 项目配置
- `src-tauri/tauri.conf.json` — Tauri 配置（窗口 1040×720、隐藏标题栏）
- `src-tauri/src/main.rs` — 入口，注册所有 command
- `src-tauri/src/` — 模块目录

**Rust 依赖：**

- `tauri` v2 — 核心框架
- `serde` / `serde_json` — 序列化
- `reqwest` — HTTP 请求（companion 通信）
- `mdns-sd` — Bonjour/mDNS 发现（配对用）
- `base64` — 图标编码
- `dirs` — userData 路径

**前端依赖：**

- `@tauri-apps/api` — Tauri 前端 API
- `@tauri-apps/cli` — 开发工具

**保留：** Vue 3 + TailwindCSS + 所有组件不变

---

## Phase 2: Rust 后端 — ADB 操作

**文件：** `src-tauri/src/adb.rs`

封装所有 ADB 命令调用（用 `std::process::Command`）：

| 函数                             | 对应 Electron         | 说明                              |
| -------------------------------- | --------------------- | --------------------------------- |
| `list_devices()`                 | `list-devices`        | 执行 `adb devices -l`，解析输出   |
| `get_device_info(serial)`        | `get-device-info`     | 8 个并行 shell 命令               |
| `disconnect_device(serial)`      | `disconnect-device`   | `adb disconnect`                  |
| `start_scrcpy(serial, opts)`     | `start-scrcpy`        | 启动 scrcpy 进程                  |
| `stop_scrcpy()`                  | `stop-scrcpy`         | 杀掉 scrcpy 进程                  |
| `launch_app(serial, pkg)`        | `launch-app`          | `adb shell am start`              |
| `install_companion(serial, apk)` | `install-companion`   | `adb install -r -g` + appops 授权 |
| `uninstall_companion(serial)`    | `uninstall-companion` | `adb uninstall`                   |
| `get_adb_path()`                 | 内部                  | 返回 bundled adb 路径             |

**注意：** scrcpy 进程退出通知用 `app.emit("scrcpy-stopped")` + 前端 `listen()` 替代 Electron 的 `ipcRenderer.on`。

---

## Phase 3: Rust 后端 — 配对（Bonjour + ADB Pair）

**文件：** `src-tauri/src/pairing.rs`

使用 `mdns-sd` crate 实现：

1. 生成随机 6 位密码 + SSID
2. 用 `uqr` 等效 Rust 库渲染 QR（或前端渲染，后端只返回 QR 内容字符串）
3. 启动 mDNS 发现 `adb-tls-pairing` 服务
4. 发现后执行 `adb pair <host>:<port> <password>`
5. 返回配对状态（idle/waiting/pairing/success/error）

**QR 渲染：** 改为前端渲染（传 `qrContent` 字符串给前端，前端用 canvas 生成 QR 图片），避免 Rust 端引入 QR 库。

---

## Phase 4: Rust 后端 — 应用列表 + 缓存

**文件：** `src-tauri/src/app.rs`, `src-tauri/src/cache.rs`

| 功能                        | 实现                                           |
| --------------------------- | ---------------------------------------------- |
| `list_apps(serial, force)`  | 从 companion HTTP 获取 + 内存/磁盘缓存         |
| `get_app_icon(serial, pkg)` | 从 companion HTTP 获取 + 磁盘缓存              |
| `clear_app_cache(serial)`   | 删除缓存文件 + 清内存                          |
| 缓存持久化                  | 写入 `app.get_path("userData")/app-cache.json` |
| 图标磁盘缓存                | 写入 `app.getPath("userData")/app-icons/`      |

---

## Phase 5: Tauri Command 注册 + 前端适配

**文件：** `src-tauri/src/main.rs`

注册所有 `#[tauri::command]`，映射关系：

```
list-devices         → list_devices
get-device-info      → get_device_info
generate-pairing-qr  → generate_pairing_qr
get-pairing-status   → get_pairing_status
disconnect-device    → disconnect_device
start-scrcpy         → start_scrcpy
stop-scrcpy          → stop_scrcpy
list-apps            → list_apps
get-app-icon         → get_app_icon
launch-app           → launch_app
install-companion    → install_companion
uninstall-companion  → uninstall_companion
clear-app-cache      → clear_app_cache
```

**前端适配层：** 创建 `src/renderer/src/api.ts`，封装 Tauri `invoke` 调用，保持与原 `window.api` 相同签名：

```ts
import { invoke } from '@tauri-apps/api/core'
export const api = {
  listDevices: () => invoke('list_devices'),
  generatePairingQr: () => invoke('generate_pairing_qr')
  // ... 所有 13 个 handle 方法
}
```

`App.vue` 和 `AppList.vue` 中的 `window.api.xxx` 改为 `import { api } from './api'` + `api.xxx`。

**窗口大小：** 用 `appWindow.setSize()` 替代 `resize-window` IPC。

---

## Phase 6: Scrcpy 迁移

**文件：** `src-tauri/src/scrcpy.rs`

- `start_scrcpy`: 用 `std::process::Command` 启动 scrcpy
- 进程退出：`app.emit("scrcpy-stopped")` 触发前端事件
- 前端用 `listen("scrcpy-stopped", ...)` 替代 `onScrcpyStopped`

---

## Phase 7: 构建与打包

**`tauri.conf.json` 配置：**

- `bundle.active: true`
- `bundle.targets: ["dmg", "app"]` (macOS)
- `bundle.identifier: "com.droidlink.app"`
- `bundle.icon: ["icons/icon.icns"]`
- `window.width: 1040, height: 720, decorations: false`

**打包命令：** `npm run tauri build`

**输出：** macOS `.app` 约 10-15MB（不含 adb/scrcpy 二进制）

## 验证方式

1. `cargo tauri dev` 启动开发模式
2. 连接手机 → 设备列表正确显示
3. 点击配对新设备 → QR 码生成 → 扫码配对成功 → 设备出现
4. 应用列表 Tab → 加载应用名称和图标
5. 屏幕镜像 → scrcpy 启动和停止正常
6. 移除设备 → 设备消失，QR 码重新出现
7. `cargo tauri build` → 生成 .app，大小 < 20MB
