# Device List Feature for Pairing UI

## Goal
Add a discovered mDNS device list next to the QR code, letting users choose which device to pair with instead of auto-pairing to the first one found.

- **Left**: QR code + instructions (existing)
- **Right**: Device list panel (new)

## Backend Changes (`src-tauri/src/pairing.rs`)

### New structs
```rust
#[derive(Serialize, Deserialize, Clone)]
pub struct DiscoveredDevice {
    pub name: String,      // mDNS service name
    pub address: String,   // ip:port
}

pub struct PairingState {
    pub status: Arc<Mutex<PairingStatus>>,
    pub devices: Arc<Mutex<Vec<DiscoveredDevice>>>,
    pub password: Mutex<Option<String>>,
}
```

### Modified/new commands

1. **`start_pairing(password)`** — Store password, reset state + devices, spawn mDNS discovery thread
   - Thread browses `_adb-tls-pairing._tcp.local.` continuously
   - On `ServiceResolved`: extract name + ip:port, add to `devices` list (deduplicate by address)
   - Does NOT auto-pair
   - 120s timeout → set status to error

2. **`get_discovered_devices()`** — Return cloned `Vec<DiscoveredDevice>`

3. **`pair_device(address: String)`** — Take stored password, run `adb pair <address> <password>` with 30s timeout
   - Set status to "pairing" while running
   - On success → "success"
   - On failure/timeout → "error"

4. **`cancel_pairing()`** — Signal the mDNS thread to stop, reset state

### mDNS thread
- Use `Arc<AtomicBool>` as cancel signal
- Loop on `browser.recv_timeout(120s)`, checking cancel flag each iteration
- On `ServiceResolved`: add device, continue loop (don't break)
- On timeout or cancel: exit thread

## Frontend Changes

### `usePairing.ts`
- Add `discoveredDevices` ref (`DiscoveredDevice[]`)
- Poll `get_discovered_devices()` alongside `get_pairing_status()` every 1s
- Add `pairDevice(address)` that invokes `pair_device`
- Add `cancelPairing()` that invokes `cancel_pairing`
- Stop polling devices when status = success/error

### `PairingQr.vue`
- Two-column layout: `flex gap-5`
- Left column: existing QR code + instructions (unchanged)
- Right column: device list panel
  - Header: "发现的设备"
  - List of device cards (follows existing DeviceCard styling pattern)
  - Each card: device name, address, type badge ("无线"), "连接" button
  - Empty state: loading spinner + "正在搜索设备..."
  - Status message at bottom (from `pairMessage`)
